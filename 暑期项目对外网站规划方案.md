# 暑期项目对外网站规划方案

更新时间：2026-03-26

## 1. 项目背景

当前暑期项目已经整理出一批项目计划书，并已挂载到云端，现有站点以总览页为主，具备基础展示能力，但还存在以下问题：

- 只有总览页，没有稳定的项目子链接
- 项目卡片与独立项目介绍页未完全打通
- 项目标签、适合年级、专业方向、人数、天数等信息未统一结构化
- 下载资料格式不统一，既有 `PDF` 也有 `DOCX`
- 项目二维码、师资介绍、筛选器、图片素材尚未系统化

因此，本阶段目标不是重做一个复杂平台，而是在现有静态站点基础上，快速做出一套可对外展示、可筛选、可下载、可分享的暑期项目网站。

## 2. 建设目标

对外网站需要满足以下目标：

- 家长、学生、合作学校点进来后，能快速看懂项目内容
- 每个项目都能进入独立详情页
- 每个项目详情页都能下载项目计划书
- 首页支持按标签筛选，快速找到合适项目
- 每个项目都能生成一个可分享的二维码
- 项目展示信息统一，避免命名、标签、难度口径不一致
- 网站适合当前云端静态部署方式，减少后端开发成本

## 3. 目标用户

- 家长：重点看适合年级、项目方向、时间长度、成果展示、安全性与价值
- 学生：重点看项目有趣程度、技能收获、是否能做成作品
- 学校或合作方：重点看课程专业度、课时安排、项目覆盖方向、师资情况
- 招生顾问或内部老师：重点看筛选效率、链接分发、二维码分享与资料下载

## 6. 网站定位与展示方式

网站建议定位为：

> 暑期项目对外招生与项目展示站

核心不是“资讯官网”，而是“项目目录 + 项目详情 + 资料下载 + 标签筛选 + 二维码分享”。

建议采用静态站点方案，原因如下：

- 当前已经挂载到云端，适合快速上线
- 项目内容以展示为主，不依赖复杂后台
- 后续新增项目可通过更新数据文件批量生成
- 二维码和子链接更容易统一维护

## 7. 信息架构

建议网站分为 5 个主要模块。

### 7.1 首页

首页承担“快速理解 + 筛选 + 进入详情”的作用，应包含：

- 网站主视觉与暑期项目定位说明
- 总项目数、纯软/软硬项目数、适合年级分布
- 项目筛选区
- 项目卡片列表
- 师资简介入口
- 常见问题或报名咨询入口

### 7.2 项目列表区

每个项目以卡片形式展示，卡片建议包含：

- 项目名称
- 一句话简介
- 项目类型：纯软 / 软硬
- 适合年级
- 专业方向
- 人数
- 天数
- 标签
- 点击查看详情
- 下载计划书

### 7.3 项目详情页

每个项目必须有独立子链接，例如：

- `/projects/vibe-coding-advanced/`
- `/projects/openclaw-camp/`
- `/projects/emotion-early-intervention/`

详情页建议统一结构：

1. 项目标题区
2. 适合对象与基础要求
3. 项目亮点
4. 学习目标
5. 课程安排与课时说明
6. 课程大纲
7. 技术栈 / 工具链
8. 项目成果展示
9. 计划书下载
10. 项目二维码
11. 相关项目推荐

### 7.4 师资页

师资页建议展示：

- 老师姓名
- 擅长方向
- 负责项目
- 代表成果或教学经历
- 联系或咨询入口

### 7.5 PDF / 下载区

所有项目详情页都应提供：

- 在线查看计划书
- 下载 PDF
- 后续如有需要，可增加源文件下载权限控制

## 9. 标签与筛选体系

标签是这次网站的核心能力，必须结构化设计，不能只在文案里零散出现。

### 9.1 基础筛选项

- 项目类型：纯软 / 软硬
- 适合年级：5-8 / 9-11
- 年龄范围：可映射到实际年龄段
- 项目方向：AI编程、视觉、生物医学、嵌入式、机器人、电子机械、环保等
- 天数：5天
- 人数：4 / 5 / 6
- 基础要求：零基础 / 有编程基础

### 9.2 扩展标签建议

- 难度：入门 / 进阶 / 高阶
- 输出成果：网页、App、硬件装置、检测系统、机器人
- 应用场景：医疗健康、无障碍、环保、陪伴、心理、教育
- 核心技能：Python、前端、MicroPython、建模、传感器、AI模型
- 兴趣导向：机器人、宠物、医疗、环境、创意装置

### 9.3 标签呈现方式

建议标签分为两层：

- 筛选标签：用于用户筛选
- 展示标签：用于项目卡片和详情页视觉表达

首页默认展示常用筛选项，详情页中可补充完整标签集和“思维导图式关联标签”。

## 12. 页面与资源目录建议

如果继续沿用静态站点，建议目录结构如下：

```text
/index.html
/projects/
  /vibe-coding-advanced/index.html
  /vibe-coding-starter/index.html
  /openclaw-camp/index.html
  ...
/teachers/index.html
/assets/
  /data/projects.json
  /data/teachers.json
  /images/projects/
  /images/teachers/
  /pdfs/
  /qrcodes/
```

这样做的好处是：

- 首页和详情页都可以从同一份 `projects.json` 读取数据
- 新增项目时只需要补数据和模板
- 二维码链接、下载地址、筛选字段可以统一维护

## 13. 页面文案和视觉建议

对外站点建议统一表达风格，不再用内部汇总式口径。

### 13.1 文案原则

- 面向家长和学生，语言清楚，不堆技术黑话
- 每个项目先讲“做什么”和“为什么有意思”，再讲技术
- 同一个字段在不同项目中用同一口径
- 避免“课程方案书”“计划书”“技术书”混用

### 13.2 视觉原则

- 首页偏招生展示风格，强调可读性和信任感
- 项目卡片突出标签、年级、方向和下载入口
- 项目详情页突出成果图、课程大纲和应用场景
- 师资页突出真实经历与对应项目方向

## 14. 实施分阶段建议

### 阶段 1：资料与口径统一

- 锁定最终项目名单
- 统一项目命名
- 补齐缺失计划书
- 拆分 Vibe Coding 高级班 / 初级版
- 整理师资介绍
- 统一 PDF 文件名

### 阶段 2：数据结构化

- 建立 `projects.json`
- 建立 `teachers.json`
- 为每个项目补齐标签、年级、方向、人数、天数、难度、基础要求
- 为每个项目确定唯一 slug

### 阶段 3：页面建设

- 首页增加筛选区
- 接入项目卡片点击跳转
- 生成项目详情页模板
- 接入下载按钮
- 接入师资模块

### 阶段 4：二维码与优化

- 基于正式子链接批量生成二维码
- 检查移动端展示
- 增加相关推荐
- 优化图片加载和 PDF 打开体验

## 15. 建议的最小可上线版本

如果需要尽快上线，建议先做一个 `MVP` 版本，包含以下内容：

- 首页项目卡片
- 基础筛选器
- 每个项目一个详情页
- 每个详情页一个 PDF 下载按钮
- 每个项目一个二维码
- 一个简化师资页

MVP 暂时可以不做：

- 复杂搜索
- 在线报名系统
- 账户体系
- 后台管理系统

## 16. 当前待确认事项

在正式开发前，建议先确认以下事项：

1. 暑期项目最终名单是否为 17 个，还是要把 `闪电小智 AI 宠物狗` 也纳入
2. `智能行宠` 与 `智能桌宠` 是否为两个不同项目
3. `Vibe Coding` 是否必须拆分为高级班和初级版两个独立详情页
4. `人型机器人`、`智能花盆`、`智能药盒` 的项目计划书由谁补齐
5. 对外正式域名或云端访问主地址是什么
6. 是否要求全部项目详情页都带图片和二维码后再上线
7. 师资介绍是否已有统一版本

## 17. 结论

本次暑期项目网站建议以“静态展示站 + 结构化项目数据”的方式推进，在现有总览页基础上扩展，不建议一开始就做复杂后台。

近期最重要的事情不是继续堆页面，而是先完成三件事：

- 锁定项目名单与命名
- 补齐缺失项目资料与 Vibe Coding 拆分
- 建立统一的项目主数据表

只要这三件事完成，后续子链接、筛选、二维码、下载页都可以稳定落地。
